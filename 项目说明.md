## Kobe Dev Studio 官方门户项目计划书

### 1. 项目概述

- **项目名称**：Kobe Dev Studio 官方 Landing Page
- **项目目标**：
  1. **合规证明**：作为自营业（个人事业主）的事业内容证明材料，提交至区役所。
  2. **品牌展示**：展示全栈开发实力及核心 SaaS 产品（保育园排班系统）。
  3. **业务窗口**：提供基本联系方式与事业者基本信息。

### 2. 核心功能模块 (MVP)

为了最快通过审查，网页应包含以下部分：

- **Hero Section**：简洁的屋号（Kobe Dev Studio）及 Slogan。
- **Service & Product**：
  - **保育园排班管理 SaaS**：重点描述针对神户本地保育规则的适配。
  - **全栈定制开发**：展示 Vue.js + Python/FastAPI 的技术栈优势。
- **About / Specific Information**：**（过关最关键部分）**
  - 屋号、代表者姓名、所在地（神户市）、业务开始日期。
- **Contact**：简单的电子邮箱或联系表单。

### 3. 技术栈选择 (Tech Stack)

- **前端框架**：原生 HTML 即可
- **UI 库**：**Tailwind CSS**（快速构建极简、专业的 Kiosk 风格或现代风格 UI）。
- **部署平台**：**GitHub Pages**。





## Kobe Dev Studio 网页制作的UI设计方案

### 1. 视觉风格指南 (Visual Identity)

- **配色方案**：建议使用“深色模式”或“高对比度现代风”。
  - *主色*：深石墨色 `#1A1A1A` 或 极客蓝 `#007AFF`。
  - *辅助色*：代码亮绿色 `#32D74B`（用于按钮或状态标识）。
- **字体**：使用等宽字体（Monospace）展示技术细节，搭配无衬线字体（Sans-serif）展示业务逻辑。
  - 示例：`JetBrains Mono` 或 `Inter`。
- **UI 元素**：加入微弱的网格背景（Grid Background）或代码片段装饰，体现“正在建设中的 SaaS”这种生命力。

### 4. GitHub Pages 部署的最后检查 (Checklist)

1. **CNAME 或 Base 路径**：
   - 如果你使用 `username.github.io/kobe-dev-studio/`，请确保 `vite.config.ts` 中的 `base` 属性已配置。
2. **No-Jekyll**：
   - 在 `public` 文件夹下放一个空的 `.nojekyll` 文件，防止 GitHub Pages 忽略以下划线开头的文件夹（对 Vue 项目很重要）。
3. **Responsive Check**：
   - 区役所职员可能会用他们的手机或老旧的平板打开你的网页，务必确保在小屏幕下地址栏和姓名依然清晰可读。
